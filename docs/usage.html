Start here

What is this project?

You drop in a bank-statement PDF. The project reads every transaction, cleans it, labels it (food, transport, salary…), and flags unusually large spends. From there it does two things:

  • Path A — instant report: a tidy Excel report, automated with n8n.
  • Path B — analytics warehouse: transactions in BigQuery, modelled with dbt, shown in a dashboard, and queryable in plain English.
Tip

You don't need all of it. Just want the Excel report? Do steps 2, 3, A. Want the warehouse + AI? Do 2, 3, C, F.

A
Use it · Path A

Generate the Excel report

The simplest thing: turn a PDF into a formatted Excel report on your own machine.

  1. Run the analyser (defaults to Account Statement.pdf if no path is given).
    bash
    python Bank_Statement_Analyser.py "your statement.pdf"
  2. Open the result — Bank_Statement_Report.xlsx — for 6 sheets: summary, transactions, monthly, categories, top merchants, anomalies.
Edit categories

Change spending categories in dbt_bank/seeds/category_keywords.csv — one file that both the report and the warehouse read.

B
Use it · Path A

n8n automation (auto-report from Drive)

Drop a PDF in Google Drive → get the report on Telegram & Email automatically.

  1. Start the Flask server (the bridge n8n calls).
    bash
    python server.py
  2. Start n8n and import the workflow.
    bash
    npm install -g n8n
n8n start
    Open http://localhost:5678 → import workflow_automation.json.
  3. Add credentials in n8n: Google Drive (OAuth2), Telegram (bot token + chat ID), Email (SMTP / Gmail app password).
  4. Create Drive folders: Bank Statements (upload here) and Bank Statement Reports (output).
  5. Publish the workflow, drop a PDF in the folder, and watch the report arrive.
C
Use it · Path B

BigQuery + dbt warehouse

Load transactions into BigQuery and model them into a tested star schema.

  1. Authenticate to Google Cloud (once).
    bash
    gcloud auth application-default login
  2. Set the project + region (your own values).
    powershell
    $env:GCP_PROJECT="n8n-upi-tracker"
$env:BQ_LOCATION="asia-south1"
$env:PYTHONIOENCODING="utf-8"
  3. Load a statement into raw.bank_transactions.
    bash
    python load_to_bigquery.py "your statement.pdf"
  4. Build + test the models (from the dbt venv).
    bash
    # from the dbt_bank/ folder
dbt debug          # check the connection
dbt build          # seeds + models + tests (fct is incremental)
Two things that bite everyone

(1) The raw dataset, the dbt profile, and the loader must all use the same region (BQ_LOCATION). (2) The incremental model uses a SQL MERGE, which needs BigQuery billing enabled — the free sandbox blocks it.

Re-score after a big change

dbt build --full-refresh rebuilds the fact table from scratch.

D
Use it · Path B

Airflow (run on a schedule)

The DAG (airflow/dags/bank_statement_pipeline.py) chains: ingest → load → dbt build → notify.

You usually don't need it to run

The DAG just automates the commands in section C — run them by hand and you've done the same thing. Airflow doesn't run natively on Windows (needs WSL2 or Docker, often blocked on corporate machines). See airflow/README.md.

Use it

Dashboard (Data Studio)

  1. Open datastudio.google.com → Blank report → BigQuery → your project → analytics dataset.
  2. Add charts from the marts: KPI scorecards + monthly income/expense (agg_monthly_summary), spend by category (agg_category_spend), top merchants & anomalies (fct_transactions).
Don't share real data

A dashboard of real transactions shouldn't be public — screenshot it blurred, or rebuild on a sample statement.

F
Use it

AI assistant (ask in plain English)

Two AI features run on the warehouse, on Claude (paid) or a free Google Gemini key — the code auto-picks whichever key you set.

Get a free key (₹0)

  1. Get a free Gemini key (Google account, no card) at aistudio.google.com/app/apikey.
  2. Install the library and set the key.
    powershell
    pip install google-genai
$env:GEMINI_API_KEY="your-free-key"
$env:GCP_PROJECT="n8n-upi-tracker"; $env:BQ_LOCATION="asia-south1"
Prefer Claude?

Set ANTHROPIC_API_KEY instead (pip install anthropic). Auto-detected; everything else is identical.

Three ways to ask

  1. Web UI — start the server, open the page:
    bash
    python server.py            # then open http://localhost:5050/
  2. Command line:
    bash
    python ai/ask_statement.py "how much did I spend on food, by month?"
  3. HTTP endpoint (for n8n / apps): POST /ask with {"question": "..."}.

Explain the anomalies

bash
python ai/explain_anomalies.py            # print plain-English reasons
python ai/explain_anomalies.py --write    # also save to a BigQuery table
Safety wall

The assistant can only run read-only SELECT queries on the analytics dataset, under a scan-size cap — it can read and reason over your data but can never change or delete anything.

G
Use it

MCP — let any AI use your tools

MCP (Model Context Protocol) is an open standard that lets AI assistants call your code directly as tools — like giving Claude or ChatGPT a set of dedicated buttons for your project, instead of having them guess from the terminal.

This project ships a ready-to-run MCP server (mcp_server.py) with three tools:

ToolWhat it does
project_statusCheck env health — which API keys are set, BQ config, output paths
analyse_pdfParse a bank-statement PDF → generate the Excel report
ask_warehouseAsk a plain-English question → runs read-only SQL on your BigQuery marts
Safety first

analyse_pdf only accepts PDFs inside a configured allowlist folder. BigQuery writes are disabled by default — the AI can read and reason over your data but cannot change anything unless you explicitly opt in.

Option A — Claude Code (direct)

If you use Claude Code, the server is registered globally and starts automatically. You can ask Claude Code things like "what is my total spend by category?" and it will query your BigQuery warehouse and reply.

bash
# Register once (user scope — works in every project)
claude mcp add -s user \
  -e GEMINI_API_KEY=your-key \
  -e GCP_PROJECT=your-project \
  -e BQ_LOCATION=asia-south1 \
  bank-statement \
  -- \
  /path/to/bsa-venv/bin/python /path/to/Bank-Statement-Analyser/mcp_server.py

# Verify
claude mcp list

Option B — OpenClaw (any AI client)

OpenClaw is a local gateway that bridges MCP servers to any AI client — Claude, ChatGPT, Gemini, Telegram bots, or web dashboards. Use this if you want to connect clients other than Claude Code.

  1. Install OpenClaw and start the gateway:
    bash
    npm install -g openclaw
openclaw gateway install
openclaw gateway start
  2. Register the bank-statement server:
    bash
    openclaw mcp set bank-statement '{
  "command": "/path/to/bsa-venv/bin/python",
  "args": ["/path/to/Bank-Statement-Analyser/mcp_server.py"],
  "cwd": "/path/to/Bank-Statement-Analyser",
  "env": {
    "GEMINI_API_KEY": "your-key",
    "GCP_PROJECT": "your-project",
    "BQ_LOCATION": "asia-south1"
  }
}'
  3. Verify all 3 tools are visible:
    bash
    openclaw mcp probe    # should show: bank-statement: 3 tools
  4. Connect your AI client to the OpenClaw gateway at ws://127.0.0.1:18789 and start asking questions.

Optional: enable BigQuery writes

The load_pdf_to_bigquery tool is hidden by default. Enable it only when you want to append parsed rows to the warehouse:

bash
export BANK_MCP_ENABLE_BQ_LOAD=true
python mcp_server.py
Cost warning

BigQuery append loads consume quota and may incur cost. Only enable this when intentionally loading new statements into the warehouse.

Troubleshooting

“No transactions found”
The PDF is a scanned image. Use a text-based PDF (you can select/copy its text).
dbt fails with a region / location error
Make BQ_LOCATION match the region of the raw dataset.
“Billing has not been enabled … DML not allowed”
Enable billing on the GCP project — the incremental model needs it. Stays within the free tier.
AI says “No LLM key found”
Set GEMINI_API_KEY (free) or ANTHROPIC_API_KEY.
Emoji / unicode error in the terminal (Windows)
Set $env:PYTHONIOENCODING="utf-8" before running.
“can't open file … \ai\ask_statement.py”
You're not in the project folder. cd into the repo first, then run:
powershell
cd C:\GIT_Projects\Bank-Statement-Analyser
python ai\ask_statement.py "your question"
dbt venv broken — “No Python at …\Python312\python.exe”

The Python 3.12 the venv was built on is gone (moved or uninstalled). Reinstall Python 3.12 (or 3.13) from python.org, then recreate the venv:

powershell
Remove-Item -Recurse -Force $HOME\dbt-venv
& "$env:LOCALAPPDATA\Programs\Python\Python312\python.exe" -m venv $HOME\dbt-venv
$HOME\dbt-venv\Scripts\Activate.ps1
pip install dbt-bigquery
dbt --version          # confirm it works

Keep dbt and the AI library in separate environments — installing the AI library (google-genai) into the dbt venv can clash with dbt's dependencies.

Glossary

ELTLoad raw data first, then clean/shape it inside the database (dbt does the “T”).
Star schemaA central facts table + small lookup tables (dimensions) around it.
Incremental modelOnly adds new rows each run instead of rebuilding everything.
Snapshot (SCD2)Keeps history — records when a value changed, with from/to dates.
Agentic AIThe model decides to use a tool, sees the result, and continues until it can answer.
CIA robot that re-runs the tests on every change so problems are caught early.
