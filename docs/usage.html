Bank Statement Analyser — Usage Guide

Everything a new person needs to run this project end to end: turn a bank-statement PDF into an Excel report, a tested BigQuery warehouse, a dashboard, and ask it questions in plain English.

PythonBigQuerydbt Airflown8nLooker Studio Claude / Gemini
Start here

What is this project?

You drop in a bank-statement PDF. The project reads every transaction, cleans it, labels it (food, transport, salary…), and flags unusually large spends. From there it does two things:

  • Path A — instant report: emails you a tidy Excel report (automated with n8n).
  • Path B — analytics warehouse: stores your transactions in BigQuery, models them with dbt, shows charts in a dashboard, and lets you ask questions in plain English.
💡 You don't need all of it. Each part works on its own. Want just the Excel report? Do sections 1, 2, A. Want the warehouse + AI? Do 1, 2, C, F.

How it fits together

One parser feeds two paths. Both share the same code and the same category rules.

2 · Install

  1. Get the code. 
    git clone https://github.com/ARAVINDHRAJA123/Bank-Statement-Analyser.git

  2. Install Python dependencies. 
    pip install -r requirements.txt
  3. (Path B only) Set up dbt in its own venv. 
    python -m venv ~/dbt-venv
~/dbt-venv/Scripts/Activate.ps1          # Windows PowerShell
pip install dbt-bigquery
Use it · Path A

A · Generate the Excel report

The simplest thing: turn a PDF into a formatted Excel report on your own machine.

  1. Run the analyser on your statement (defaults to Account Statement.pdf if no path given). 
    python Bank_Statement_Analyser.py "your statement.pdf"
  2. Open the result — Bank_Statement_Report.xlsx — for 6 sheets: summary, transactions, monthly, categories, top merchants, and anomalies.
💡 Edit spending categories in dbt_bank/seeds/category_keywords.csv — one file that both the report and the warehouse read.

B · n8n automation (auto-report from Google Drive)

Drop a PDF in Google Drive → get the report on Telegram & Email automatically.

  1. Start the Flask server (the bridge n8n calls). 
    python server.py
  2. Start n8n and import the workflow. 
    npm install -g n8n
n8n start
    Open http://localhost:5678 → import workflow_automation.json.
  3. Add credentials in n8n: Google Drive (OAuth2), Telegram (bot token + chat ID), Email (SMTP / Gmail app password).
  4. Create Drive folders: Bank Statements (upload here) and Bank Statement Reports (output).
  5. Publish the workflow, then drop a PDF in the Drive folder and watch the report arrive.
Use it · Path B

C · BigQuery + dbt warehouse

Load transactions into BigQuery and model them into a tested star schema.

  1. Authenticate to Google Cloud (once). 
    gcloud auth application-default login
  2. Set the project + region (use your own values). 
    $env:GCP_PROJECT="n8n-upi-tracker"
$env:BQ_LOCATION="asia-south1"
$env:PYTHONIOENCODING="utf-8"
  3. Load a statement into raw.bank_transactions. 
    python load_to_bigquery.py "your statement.pdf"
  4. Build + test the models (from the dbt venv). 
    cd dbt_bank
dbt debug          # check the connection
dbt build          # seeds + models + tests (fct_transactions is incremental)
⚠️ Two things that bite everyone. (1) The raw dataset, the dbt profile, and the loader must all use the same region (BQ_LOCATION). (2) The incremental model uses a SQL MERGE, which needs BigQuery billing enabled — the free sandbox blocks it.
💡 Re-score after a big data change: dbt build --full-refresh rebuilds the fact table from scratch.

D · Airflow (run Path B on a schedule)

The DAG (airflow/dags/bank_statement_pipeline.py) chains: ingest → load → dbt build → notify.

💡 You usually don't need Airflow to run. The DAG just automates the commands in section C. Run them by hand and you've done exactly what it does. Airflow doesn't run natively on Windows (needs WSL2 or Docker — often blocked on corporate machines). See airflow/README.md.

E · Dashboard (Looker Studio)

  1. Open lookerstudio.google.com → Blank report → BigQuery connector → your project → analytics dataset.
  2. Add charts from the marts: KPI scorecards + monthly income/expense (agg_monthly_summary), spend by category (agg_category_spend), top merchants & anomalies (fct_transactions).
⚠️ Don't share a dashboard of real transactions publicly. Screenshot it blurred, or rebuild on a sample statement.

F · AI assistant (ask questions in plain English)

Two AI features run on the warehouse, on Claude (paid) or a free Google Gemini key — the code auto-picks whichever key you set.

Get a free key (₹0)

  1. Get a free Gemini key (Google account, no card) at aistudio.google.com/app/apikey.
  2. Install the library and set the key. 
    pip install google-generativeai
$env:GEMINI_API_KEY="your-free-key"
$env:GCP_PROJECT="n8n-upi-tracker"; $env:BQ_LOCATION="asia-south1"
💡 Prefer Claude? Set ANTHROPIC_API_KEY instead (pip install anthropic). It's auto-detected; everything else is identical.

Three ways to ask

  1. Web UI — start the server and open the page: 
    python server.py            # then open http://localhost:5050/
  2. Command line: 
    python ai/ask_statement.py "how much did I spend on food, by month?"
  3. HTTP endpoint (for n8n / other apps): POST /ask with {"question": "..."}.

Explain the anomalies

python ai/explain_anomalies.py            # print plain-English reasons
python ai/explain_anomalies.py --write    # also save to a BigQuery table
✅ Safety wall. The assistant can only run read-only SELECT queries on the analytics dataset, under a scan-size cap — it can read and reason over your data but can never change or delete anything.

Tests & CI

Run the test suite anytime:

python -m pytest -q

Every push to GitHub also runs the tests and validates the dbt models automatically (GitHub Actions). The green “CI passing” badge on the repo means everything still works.

Interactive dbt docs

Browse the model lineage and column docs online (GitHub Pages) or locally:

cd dbt_bank
dbt docs generate
dbt docs serve         # opens an interactive lineage graph

Troubleshooting

SymptomFix
“No transactions found”The PDF is a scanned image. Use a text-based PDF (you can select/copy its text).
dbt fails with a region/location errorMake BQ_LOCATION match the region of the raw dataset.
“Billing has not been enabled … DML not allowed”Enable billing on the GCP project — the incremental model needs it. Stays within the free tier.
AI says “No LLM key found”Set GEMINI_API_KEY (free) or ANTHROPIC_API_KEY.
Emoji/unicode error in the terminal (Windows)Set $env:PYTHONIOENCODING="utf-8".

Glossary

ELTLoad raw data first, then clean/shape it inside the database (dbt does the “T”).
Star schemaA central facts table + small lookup tables (dimensions) around it.
Incremental modelOnly adds new rows on each run instead of rebuilding everything.
Snapshot (SCD2)Keeps history — records when a value changed, with from/to dates.
Agentic AIThe model decides to use a tool, sees the result, and continues until it can answer.
CIA robot that re-runs the tests on every change so problems are caught early.
